Unsupervised Error Log Clusterization for ATLAS Computing
Key Objectives of the Research

During the last decade, data centers and computer networks have grown significantly in processing power, size, and complexity. Large-scale distributed systems are widespread, supporting millions of concurrent users from various application areas. This type of infrastructure provides common, friendly and flexible interfaces, but also is subject to many dynamic failures. Error logs are a fruitful source of information both for diagnosis as well as for proactive fault handling â€“ however the large size and complexity of these systems makes diagnosing failures a challenging problem. As a result, organizations commonly have to handle many gigabytes of log data on a daily basis. Network faults, service failures, security incidents, and other error conditions often trigger log messages which provide detailed error information to system administrators.

In order to analyze large amounts of textual log data without well-defined structure, several data mining methods have been proposed in the past which focus on the detection of line patterns from textual event logs. However, such tools assume that human experts interpret detected knowledge and create the rules (patterns or regular expressions) manually which is time-consuming and expensive. This approach does not allow to identify previously unknown error conditions.

Possible solution is to cluster log errors using methods of unsupervized text clusterization.

Requirements

  • Performance, suitable for the real-time applications
  • Unknown number of clusters
  • Capture outliers (previously unknown errors conditions)
  • Minimization of human intervention in the process of clusterization (automatic detection of clusterization parameters)
